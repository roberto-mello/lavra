# Lavra v0.7.7 Release Notes

**Release date:** TBD

---

## Bug fixes

### `.lavra/` paths now resolve from the git root, not wherever you happen to be standing

This one was subtle and annoying. When a subagent changed into a subdirectory during work — say, into `app/models/` to run tests — every subsequent `.lavra/` path lookup failed. The agent would try to read `.lavra/config/lavra.json` or write `.lavra/memory/session-state.md` relative to `app/models/` instead of the project root. The file wasn't there, so the operation silently no-oped.

This hit basically everything: workflow config loading, knowledge recall, session state persistence, and checkpointing. We've now added a `<project_root>` block to every command, skill, and agent that touches `.lavra/` files. The pattern is simple:

```bash
PROJECT_ROOT=$(git rev-parse --show-toplevel 2>/dev/null || echo "$PWD")
```

Then prefix every `.lavra/` path with `"$PROJECT_ROOT/"`. You can `cd` wherever you need to during work without losing the memory system.

## Improvements

### npm package excludes platform-specific generated directories

The published package no longer ships `plugins/lavra/opencode/`, `plugins/lavra/gemini/`, and `plugins/lavra/cortex/`. These are generated at install time by the conversion scripts, so including them in the tarball just bloated the download. The `package.json` `files` array now explicitly excludes them.

### Conversion scripts rewrite hook paths too

The OpenCode, Gemini, and Cortex converters were already rewriting `.claude/skills/` to the platform equivalent. They missed `.claude/hooks/`. Fixed now — all three platforms get the correct hook directory path in generated files.

### Installers require `bd init` before running

The installers used to silently run `bd init` on your behalf if `.beads/` was missing. That's overreach — you might not want beads initialized with the defaults, or you might be running the installer against the wrong directory. Now they exit with instructions:

```
[!] No .beads directory found in /your/project.

    Run this first:
      cd /your/project && bd init
```

### Missing sqlite3 triggers a warning

The memory system uses SQLite as a search index for fast recall. It always worked without it — falling back to linear JSONL search — but never told you that. Now the installer warns if `sqlite3` isn't found, with install instructions. If you've been wondering why recall feels slow on large knowledge bases, this is probably why.

### Context7 MCP server confirmed in post-install summary

The installer was already adding Context7 to your MCP config. It just never said so in the final output. Now it does. Context7 gives agents access to current framework documentation without a web search.
